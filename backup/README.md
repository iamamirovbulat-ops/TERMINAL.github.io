# Система видеотерминалов

Система для организации видеосвязи между клиентами и операторами через терминалы-киоски.

## Функционал

- **Терминал**: Страница с QR-кодом и номером телефона для инициации видеосвязи
- **Кабинет оператора**: Панель управления с уведомлениями о новых звонках
- **Видеосвязь**: WebRTC видеозвонки между терминалом и оператором
- **Интеграция с Битрикс24**: Автоматическое создание сделок при обращении

## Как это работает

1. Клиент подходит к терминалу и видит QR-код и номер телефона
2. Сканирует QR-код или звонит по номеру
3. Система автоматически создает сделку в Битрикс24
4. Оператор получает звуковое уведомление и видит новый запрос
5. Оператор принимает звонок и начинается видеосвязь
6. После завершения звонка терминал возвращается на начальную страницу

## Установка и запуск

### 1. Установка зависимостей

```bash
npm install
```

### 2. Настройка конфигурации

Откройте файл `server.js` и проверьте настройки:

```javascript
const BITRIX_CONFIG = {
  webhook: 'https://sbu.bitrix24.ru/rest/1/r76d49oc0j2vktuz/',
  domain: 'sbu.bitrix24.ru',
  pipelineId: 24
};

const PHONE_NUMBER = '89030619769';
```

### 3. Запуск сервера

```bash
npm start
```

Для разработки:
```bash
npm run dev
```

### 4. Доступ к интерфейсам

- **Терминал**: http://localhost:3000/
- **Кабинет оператора**: http://localhost:3000/operator

## Технические особенности

### Терминал
- Не имеет сенсорного экрана или физических кнопок
- Работает как "киоск-монитор" с камерой
- Автоматически возвращается на начальную страницу после звонка

### Видеосвязь
- Использует WebRTC для прямого соединения
- STUN сервер для обхода NAT
- Автоматическое определение камер и микрофонов

### Интеграция с Битрикс24
- Автоматическое создание сделки в воронке с ID 24
- Название сделки: "Обращение по терминалу #ID"
- Добавление информации о времени начала и номере телефона

### Уведомления оператора
- Звуковое оповещение при новом звонке
- Всплывающее окно с информацией о клиенте
- Возможность принять или отклонить звонок

Убедитесь что webhook имеет необходимые права для создания сделок.

## Траблшутинг

### Проблемы с видеосвязью
- Проверьте доступ к камере и микрофону
- Убедитесь что браузер поддерживает WebRTC
- Проверьте сетевые настройки и файрвол

### Проблемы с Битрикс24
- Проверьте правильность webhook URL
- Убедитесь что пользователь имеет права на создание сделок
- Проверьте ID воронки (должен быть 24)

### Проблемы с QR-кодом
- Убедитесь что URL в QR-коде доступен из сети
- Проверьте что домен правильно разрешается
